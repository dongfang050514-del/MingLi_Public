# MingLi

MingLi 是一个纯静态前端的命理 H5 项目。当前版本开始从单一八字页扩展为命理工具站：主界面负责选择功能，本命八字和小六爻作为两个独立入口运行。页面在本地完成计算与交互，不依赖后端服务。

## 当前版本

**MingLi1.1.0**

`MingLi1.0.0` 是八字页面的第一个相对完整版本，适合作为后续迭代的稳定基线。本次 `MingLi1.1.0` 加入主界面和“小六爻问事”入口，形成命理工具站的第一版双入口结构。

核心能力：

- 普通版默认流程：年份、月份、日期、出生时段、称呼、想问主题。
- 专业版校准项：城市、经度、真太阳时、年柱流派、日柱换日、具体小时分钟。
- 本地静态计算：不依赖后端服务，不上传用户输入。
- 命盘过场：出生数据飞入命盘，依次校时、排柱、归五行。
- 结果呈现：一签一页，支持左右滑动切换；结果尽量用白话解释，而不是只展示术语。
- 微信测试版：提供单文件构建，适合静态托管后在微信中打开。
- 小六爻问事：支持心中默念后直接起卦，通过铜钱落位、悬挂签牌和白话解读给出当下行动建议。
- 沉浸式结果：小六爻结果页采用问卦台、木梁、垂绳、悬挂签牌和左右滑动切换，强化手机端一屏体验。

## 功能入口

```text
index.html
```

主界面，提供本命八字和小六爻两个入口。

```text
bazi/index.html
```

本命八字页面，保留原有卡牌填写、命盘动画、结果签牌和专业版校准能力。

```text
liuyao/index.html
```

小六爻问事页面。默认支持“心中默念一事后直接起卦”，不强制用户先填写问题；用户也可以展开补充所问之事，或从常见问法中快速带入。问事方向覆盖进退选择、感情、事业、财物、学业、文书、消息、出行、失物、合作、家宅、身心状态、近期走势等日常场景，再通过六次铜钱落位生成签牌式结果。

设计原则：

- 起卦动画以铜钱、线条、问卦台和签牌为主，保留传统氛围，但减少用户必须理解的术语。
- 起卦入口以“直接起卦”为主，问题和方向只是可选补充，避免用户被表单卡住。
- 最终结果必须翻译成普通用户能理解的内容，回答“现在是什么局面、哪里容易卡、接下来怎么做”，不能只给术语。
- 结果页采用悬挂签牌，一页只展示一个重点判断，支持左右滑动切换。

## 版本规则

后续按语义化版本迭代：

- `1.0.x`：小修小改，例如文案优化、动效微调、兼容性修复。
- `1.x.0`：新增功能或新页面，例如增加“小六爻”入口、首页工具站结构。
- `2.0.0`：大版本更新，例如整体产品正式上新、信息架构重做、核心体验大改。

当前命名采用项目习惯格式：`MingLi1.1.0`。

## 文件入口

- `index.html`：主界面入口，引用 `home.css`。
- `home.css`：主界面样式。

- `app.js`：八字卡牌流程、计算、命盘动画、结果页交互。
- `liuyao/`：小六爻问事页面源码。
- `MingLi-wechat.html`：微信测试用单文件版。
- `wechat-release/index.html`：主界面静态托管入口，当前为单文件版。
- `wechat-release/bazi/index.html`：八字静态托管入口，当前为单文件版。
- `wechat-release/liuyao/index.html`：小六爻静态托管入口，当前为单文件版。
- `MingLi-wechat-release.zip`：可上传到静态托管平台的压缩包。
- `edgeone.json`：腾讯云 EdgeOne Pages 部署配置。

## 部署建议

GitHub Pages 可用于开发测试，但在微信内置浏览器中偶尔会访问不稳定。正式分享给普通用户时，更建议部署到腾讯云 EdgeOne Pages、CloudBase 或其他稳定的 HTTPS 静态托管服务。

EdgeOne Pages 建议配置：

- 框架预设：`Other`
- 根目录：`./`
- 输出目录：`wechat-release`
- 构建命令：`echo Static files are already prepared in wechat-release`
- 安装命令：`echo No dependencies required`

## 后续方向

下一阶段计划把 MingLi 从单一八字页扩展为命理工具站：

- 首页：`MingLi 命理阁`
- 入口一：`本命八字`，翻开你的本命盘。
- 入口二：`小六爻`，为眼前一事起卦。

八字用于建立长期本命档案，小六爻用于日常随手问事。过场动画可以保留传统术语和仪式感，但最终结果需要翻译成用户能理解、能行动的白话建议。
